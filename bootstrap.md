## Принципы (запомни на всю сессию)

1. **Convention, not catalog.** Резидент может захотеть что угодно. Твоя задача —
   не «выбрать пункт меню», а понять намерение и собрать по стандартам COMANDOS.
2. **Не ломай существующее.** Если проект уже есть — изучи и уважай его структуру.
3. **Один язык — TypeScript** для всех продуктов (дефолт). Python — только если
   задача требует тяжёлого ML и резидент это подтвердил.
4. **Всегда оставляй след:** в созданном проекте должен быть `AGENTS.md` и
   `PROJECT-BRIEF.md`, чтобы следующая сессия знала контекст.

## ШАГ 1. DOCTOR — выровнять окружение

Запусти `doctor/doctor.sh` (или выполни его логику вручную, если bash недоступен):

```bash
bash doctor/doctor.sh
```

Doctor делает:
1. Проверяет базовое: `node`, `git`. Чего нет — подсказывает как поставить.
2. Читает/создаёт `~/.comandos/config.json` (см. ШАГ 2).
3. Находит/привязывает базу знаний (см. ШАГ 3).

Если `doctor.sh` не запускается (нет прав/Windows) — выполни шаги 2–3 сам через диалог.

---

## ШАГ 2. Глобальный конфиг резидента

Файл `~/.comandos/config.json`. Если его НЕТ — создай, задав человеку 3 вопроса:

```json
{
  "name": "<как зовут резидента>",
  "projects_dir": "<куда складывать проекты, по умолчанию ~/Projects>",
  "knowledge_base": "<путь к базе знаний или null>",
  "ide": "<claude-code | codex | antigravity — определи сам по среде>"
}
```

Если файл ЕСТЬ — прочитай и НЕ переспрашивай. Это разовая настройка навсегда.

## ШАГ 3. База знаний (vault) — найти и привязать

1. Поищи признаки базы знаний на компе: папки с `.obsidian/`, git-репозитории
   с большим числом `.md`, типичные имена (`vault`, `Obsidian`, `MyVault`, `Второй мозг`).
2. Нашёл → спроси: **«Это твоя база знаний? <путь> — да/нет»**.
   - Да → запиши путь в `knowledge_base` конфига. Проект будет туда ссылаться.
3. Не нашёл / ответ «нет» → предложи на выбор:
   - **(a, рекомендуется)** «У тебя настроен Второй мозг из курса COMANDOS?
     Если нет — вот модуль: <TODO: ссылка на модуль>. Настрой и вернись —
     тогда все проекты будут компаундить знания.»
   - **(b)** «Создать простую базовую папку для знаний прямо сейчас?» →
     создай `<projects_dir>/_knowledge/` с `README.md`.
   - **(c)** «Пропустить — проект будет standalone.» → `knowledge_base: null`.
4. Запиши выбор в конфиг. Больше не спрашивай.

## ШАГ 4. ИНТЕРВЬЮ — что делаем

Спроси по порядку (по одному, не вываливай всё сразу):

1. **«Новый проект или дорабатываем существующий?»**
   - Существующий → перейди к ШАГУ 4-Б.
2. **«Опиши своими словами, что хочешь сделать.»**
3. По ответу определи ВЕТКУ (см. `recipes/_schema.md`):
   - Разовый артефакт (презентация/видео/картинка/гайд) → ветка A
   - Продукт (сайт/бот/агент/SaaS/mini-app) → ветка B
   - Знание (research/конспект) → ветка C
4. Подбери рецепт из `recipes/`. Если точного нет — возьми ближайший
   или собери по правилам палитры (см. `recipes/_schema.md` → «Открытый путь»).
5. Задай уточняющие вопросы из поля `ask` рецепта.

### ШАГ 4-Б. Существующий проект
1. Прочитай его `README.md`, `AGENTS.md`/`CLAUDE.md`, `package.json`.
2. Составь карту: где код, где доки, где медиа (§17). Запиши в `.comandos/project-map.md`.
3. НЕ меняй структуру — клади новое в правильные места по карте.

## ШАГ 5. РЕЦЕПТ — развернуть

В зависимости от типа рецепта (`recipes/<name>.json`):

- **`template`** (есть код-папка в kit) → скопируй её в новый проект.
- **`repo`** (готовый репо, напр. site) → `npx degit Comandosai/site-starter <target>`.
- **`action: skill`** → не создавай проект, запусти указанный навык.
- **`action: mcp`** → выполни через указанный MCP (напр. генерация видео).

Перед копированием кода прогони doctor по `needs` рецепта (CLI/MCP/env) —
доустанови недостающее. См. `lib/mcp-install.md`.

## ШАГ 6. ФИНАЛИЗАЦИЯ

1. В созданный проект положи из `conventions/`:
   - `AGENTS.md` (правила для будущих сессий) — из `AGENTS.md.tmpl`
   - `PROJECT-BRIEF.md` — из `PROJECT-BRIEF.md.tmpl`, заполни по интервью
2. Если `knowledge_base` задан — создай там запись о проекте (см. `conventions/vault-link.md`).
3. Покажи человеку: что создано, как запустить (1 команда), что дальше.
4. Спроси: «Запустить локально / задеплоить, чтобы увидеть вживую?»
