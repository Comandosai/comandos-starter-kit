# Рецепты — что это и как устроены

**Рецепт** = маленькая карточка-инструкция (JSON), которая говорит агенту, как
развернуть один тип запроса. НЕ код. Код живёт в `templates/` или во внешних репо.

## Поля рецепта

```jsonc
{
  "name": "человекочитаемое имя",
  "branch": "A | B | C",            // A=артефакт, B=продукт, C=знание
  "stack": "короткое описание стека", // только для B

  // ОТКУДА брать (ровно одно из трёх):
  "template": "templates/<dir>",      // код-папка внутри kit
  "repo": "Comandosai/<repo>",        // внешний репо (degit)
  "action": "skill | mcp",            // не код, а вызов навыка/MCP

  "skill": "имя-навыка",              // если action=skill
  "mcp_tool": "имя-инструмента",      // если action=mcp

  // ЧТО нужно окружению (doctor доустановит):
  "needs": {
    "cli": ["node"],
    "mcp": ["supabase"],
    "env": ["SOME_TOKEN"]
  },

  // ВОПРОСЫ резиденту перед сборкой:
  "ask": ["вопрос 1", "вопрос 2"]
}
```

## Две дороги внутри ветки B (продукты)

### Протоптанная дорога (paved road)
Запрос совпал с готовым рецептом → берём `template`/`repo`. ~80% случаев. Быстро,
протестировано, работает с первого раза.

### Открытый путь (agent reasons)
Точного рецепта НЕТ. Агент собирает сам по правилам палитры:

| Признак запроса | Бери |
|---|---|
| Нужен сайт/лендинг/контент-страницы | Astro + Tailwind |
| Нужен чат-бот в мессенджере | grammY (TS) |
| Нужен «цифровой сотрудник» / автономная логика на LLM | Claude Agent SDK (TS) |
| Нужны вход/БД/платежи/личный кабинет | Next.js + Supabase (+ Stripe) |
| Веб внутри Telegram | Next.js + Telegram Mini App SDK |
| Только текст/знания | markdown |
| Тяжёлый ML / спец-библиотеки только в Python | Python (escape hatch) |

Собрав по правилам — всё равно положи `AGENTS.md` + `PROJECT-BRIEF.md` и привяжи к базе знаний.

## Ветка A — разовые артефакты (НЕ создаём проект)
Презентация, видео, картинка, гайд, пост. Рецепт указывает `action: skill|mcp`.
Агент запускает готовый навык/MCP, кладёт результат по §17, фиксирует в базе знаний.
