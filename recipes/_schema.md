# Рецепты — что это и как устроены

**Рецепт** = маленькая карточка-инструкция (JSON), которая говорит агенту, как
развернуть один тип запроса. НЕ код. Код живёт в `templates/` или во внешних репо.

## Поля рецепта

```jsonc
{
  "name": "человекочитаемое имя",
  "branch": "A | B | C",            // A=артефакт, B=продукт, C=знание
  "stack": "короткое описание стека", // только для B

  // ОТКУДА брать (ровно одно из четырёх):
  "template": "templates/<dir>",      // код-папка внутри kit (скопировать)
  "repo": "Comandosai/<repo>",        // внешний репо (npx degit)
  "scaffold_cmd": "npx create-... \"<target>\"", // официальный CLI + поле "post"
  "action": "skill | mcp",            // не код, а вызов навыка/MCP

  "skill": "имя-навыка",              // если action=skill
  "mcp_tool": "имя-инструмента",      // если action=mcp (полное: mcp__<server>__<tool>)
  "post": ["доп. шаги после scaffold_cmd"],

  // ЧТО нужно окружению (агент доустановит по lib/mcp-install.md):
  "needs": {
    "cli": ["node"],
    "mcp": ["supabase"],
    "mcp_env": ["SUPABASE_ACCESS_TOKEN"],  // токены для MCP-сервера (в конфиг IDE)
    "env": ["SOME_TOKEN"]                   // ключи для приложения (в .env проекта)
  },

  "ask": ["вопрос 1", "вопрос 2"]     // спросить у резидента перед сборкой
}
```

`<target>` = `<projects_dir>/<slug-имени-проекта>`. Подстановку делает агент (ШАГ 5).

## Две дороги внутри ветки B (продукты)

### Протоптанная дорога (paved road)
Запрос совпал с готовым рецептом → берём `template`/`repo`/`scaffold_cmd`. ~80% случаев.

### Открытый путь (agent reasons) — детерминированный алгоритм
1. Точное совпадение рецепта → используем его источник.
2. Точного нет, но есть строка палитры ниже → собираем по ней.
3. Нет ни рецепта, ни строки палитры → проверяем профильный навык в среде
   (напр. `browser-extension-builder`); есть → используем; нет → честно говорим
   резиденту и предлагаем ближайший разумный стек, подтверждаем с ним.

| Признак запроса | Бери |
|---|---|
| Сайт/лендинг/контент-страницы | Astro + Tailwind (`repo` site-starter) |
| Чат-бот в мессенджере | grammY (TS) — `templates/bot` |
| «Цифровой сотрудник» / автономная логика на LLM | Claude Agent SDK (TS) — `templates/agent` |
| Вход/БД/платежи/личный кабинет | Next.js + Supabase (+Stripe) — `scaffold_cmd` |
| Веб внутри Telegram (mini app) | Next.js + Telegram Mini App SDK — `recipes/miniapp` |
| Браузерное расширение (Chrome/Firefox) | Vite + CRXJS + TS, Manifest V3 (навык browser-extension-builder) |
| Только текст/знания | markdown — `templates/research` |
| Тяжёлый ML / спец-библиотеки только в Python | Python (escape hatch, подтвердить) |

Собрав по правилам — всё равно положи `AGENTS.md` + `PROJECT-BRIEF.md` и привяжи к базе знаний.

## Ветка A — разовые артефакты (НЕ создаём проект)
Презентация, видео, картинка, гайд, пост. Рецепт указывает `action: skill|mcp`.
Агент запускает готовый навык/MCP; если навыка нет в среде — собирает артефакт
напрямую своими силами (не блокируется). Результат сохраняется в `_artifacts/`
или в базу знаний (`conventions/vault-link.md`). `AGENTS.md`/`PROJECT-BRIEF` не нужны.
