# COMANDOS Starter Kit — маркетинговая страница

## 📄 ДЕТАЛЬНАЯ СТРАНИЦА

## 😩 Знакомая боль?

Хочешь собрать проект — сайт, бота, цифрового сотрудника, — но первые часы уходят не на
дело, а на возню: какой стек, как настроить окружение, куда положить ключи, почему `git`
ругается, почему `npm` не ставится. А если ты не программист — это просто стена.

COMANDOS Starter Kit убирает эту стену. Ты говоришь, что хочешь сделать. Всё остальное
агент делает за тебя — и в конце показывает рабочий каркас с пройденными проверками.

## ✨ Что делает этот навык

Один промпт запускает мастер-сценарий из 8 фаз:

1. **Doctor** — проверяет окружение (Node, git, какая у тебя AI-среда) и сам доустанавливает недостающее.
2. **Конфиг + база знаний** — один раз настраивает профиль резидента и привязывает твой «Второй мозг».
3. **Интервью** — расспрашивает простыми словами, что нужно сделать.
4. **Рецепт** — подбирает стек: сайт (Astro), бот (grammY), цифровой сотрудник (Claude Agent SDK), Micro-SaaS (Next.js + Supabase), mini-app, презентация/видео, research.
5. **Сборка** — разворачивает структуру, конфиги, рабочий `.env`, правила для всех трёх AI-сред.
6. **Проверка** — прогоняет `typecheck` / `lint` / `test` / smoke. Не скажет «готово», пока не зелёное.
7. **Привязка** — создаёт заметку о проекте в твоей базе знаний.
8. **Итог** — что собрано, как запустить, что дальше.

## 🚀 Как установить и запустить

Открой свою AI-среду (Claude Code, Codex или Antigravity) и вставь один промпт:

```
Установи и запусти COMANDOS Starter Kit.
1) git clone https://github.com/Comandosai/comandos-starter-kit ~/.comandos-kit
   (Windows PowerShell: git clone https://github.com/Comandosai/comandos-starter-kit "$HOME\.comandos-kit")
   (если уже есть — git -C ~/.comandos-kit pull)
2) Прочитай bootstrap.md в этой папке и выполни по шагам.
Общайся по-русски, просто. Я не программист — делай всё за меня.
```

Дальше агент ведёт сам: задаёт вопросы по одному, собирает проект, прогоняет проверки и
показывает результат. От тебя — только ответы на простые вопросы.

## 🔑 Ключи и база знаний — нужны ли заранее?

Нет. Костяк собирается без ключей: smoke-проверка проходит с пустым `.env` и даёт понятную
подсказку, а не ошибку. Ключ навык спросит только тогда, когда без него выбранный рецепт
бессмыслен (например, Supabase для SaaS), и положит его в локальный `.env` проекта — он
никогда не попадает в git. База знаний опциональна: если её нет, проект собирается как
standalone, ничего не ломается.

## 🛠 Что поддерживается

| Тип проекта | Стек |
|---|---|
| Сайт / лендинг | Astro + Tailwind |
| Telegram-бот | grammY (TypeScript) |
| Цифровой сотрудник | Claude Agent SDK (TypeScript) |
| Micro-SaaS | Next.js + Supabase (+ Stripe) |
| Telegram Mini App | Next.js + Telegram Mini App SDK |
| Презентация / видео | навыки и MCP |
| Research / знание | Markdown |

Принцип «Convention, not catalog»: рецепты — это быстрые дефолты, а не ограничение. Хочешь
нестандартное — агент соберёт по правилам COMANDOS.

## 🎁 Что ты получаешь на выходе

- Профессиональный костяк проекта: структура, зависимости, конфиги, рабочий `.env`, тесты, скрипты.
- Файлы-правила для трёх сред (`AGENTS.md`, `CLAUDE.md`, `.agent/rules/`) — проект одинаково открывается в Claude Code, Codex и Antigravity.
- Первый git-коммит и пройденный гейт качества.
- Заметку о проекте в твоей базе знаний (если она привязана).
- Чёткий следующий шаг: открыть новый чат в папке проекта и наполнять его — агент уже в контексте.
