# COMANDOS Starter — оркестратор

## 🚀 КАК ТОЛЬКО навык активирован — НЕМЕДЛЕННО запусти сценарий

Этот навык — **исполняемый мастер**, а не справка. Как только он активирован
(пользователь вызвал его, сказал «создай проект», или ты только что его установил):

1. **ОТКРОЙ `bootstrap.md` в папке этого навыка и выполни его пошагово, до конца.**
   Не пересказывай его — ИСПОЛНЯЙ. Не останавливайся после установки файлов.
2. **Поздоровайся и сразу спроси простыми словами: «Что хочешь создать?»**
   (сайт, Telegram-бот, цифровой сотрудник, приложение, презентация…). Пользователь —
   НЕ программист: один вопрос за раз, без стектрейсов, всё делай за него.
3. **Пройди ВСЕ фазы по порядку, НИЧЕГО не пропуская.** Особенно — три, которые
   нельзя пропускать никогда:
   - **ИНТЕРВЬЮ** — сначала спроси, что создаём (без этого не собирай файлы).
   - **БАЗА ЗНАНИЙ (обязательно)** — найди/привяжи базу знаний. Если её нет — заведи
     простую (`<projects_dir>/_knowledge/`). НЕ делай проект standalone по умолчанию.
   - **ПРИВЯЗКА** — в конце ОБЯЗАТЕЛЬНО запиши заметку о проекте в базу знаний
     со ссылкой на репозиторий/папку с кодом (`conventions/vault-link.md`).

> Если ты сейчас только установил навык (склонировал файлы) — это НЕ конец.
> Сразу переходи к шагу 1 выше и запускай мастер. «Установлено» ≠ «сделано».

Подробный пошаговый рулбук — в `bootstrap.md`. Ниже — карта фаз.

## Поток (фазы)

```
DOCTOR → CONFIG+VAULT → ИНТЕРВЬЮ → РЕЦЕПТ → СБОРКА → ПРОВЕРКА → ПРИВЯЗКА → ИТОГ
```

1. **DOCTOR** (`doctor/doctor.sh` / `.ps1`) — окружение, IDE, версия Node, кандидаты базы знаний.
2. **CONFIG+VAULT** — `~/.comandos/config.json` (создать если нет); **обязательно** найти/
   привязать базу знаний (нет — завести простую, не пропускать); предупредить если
   `projects_dir` в облаке (iCloud/Dropbox — git/npm не работают).
3. **ИНТЕРВЬЮ** — новый/существующий проект + что делаем → ветка (A/B/C) + рецепт.
4. **РЕЦЕПТ** — развернуть: `template`/`repo`/`scaffold_cmd`/`action:skill|mcp`; доустановить `needs`.
5. **СБОРКА** — полный набор файлов в проект (`conventions/`): `AGENTS.md`+`CLAUDE.md`+
   `.agent/rules`+`PROJECT-BRIEF.md`+`.gitignore`+**рабочий `.env`**; подставить все `{{...}}`.
6. **ПРОВЕРКА** (gate) — `git init`, `npm install`, проверка типов, smoke-запуск. Не говорить
   «готово», пока не зелёное.
7. **ПРИВЯЗКА** — обязательно создать заметку о проекте в базе знаний (`conventions/vault-link.md`).
8. **ИТОГ** — что создано, результат проверки, как запустить, что дальше.

Полная самопроверка — блок «Definition of Done» в конце `bootstrap.md`.

## Правила
- **Установка ≠ запуск.** Положил файлы — сразу запускай мастер (см. блок 🚀 выше).
- **Сначала спроси, что создаём.** Не собирай файлы до интервью.
- **База знаний обязательна.** Всегда найди/заведи её и в конце запиши заметку о проекте
  со ссылкой на код. Standalone — только если резидент явно отказался.
- **Скелет кладётся всегда, когда будет код.** Если проект станет продуктом (сайт/бот/агент/
  SaaS/mini-app) — клади профессиональный каркас (папки + рабочий `.env` + конфиги под стек)
  СРАЗУ, даже если резидент говорит «сначала концепция». «Сначала концепция» = `src/` заглушки
  + `docs/CONCEPT.md`, но скелет и `.env` есть. Без скелета — только разовый артефакт и чистое
  знание без код-будущего. Сомневаешься — клади скелет или спроси прямо.
- Резидент — НЕ программист. Никаких стектрейсов. Всё делать за него, объяснять просто.
- TypeScript по умолчанию для продуктов. Не ломать существующую структуру.
- Convention, not catalog: рецепты — дефолты, а не ограничение. Хочет иное — собрать по правилам.

## Файлы kit
- `bootstrap.md` — полный пошаговый сценарий (читать первым)
- `doctor/doctor.sh` — проверка окружения
- `lib/mcp-install.md` — установка MCP/CLI по каждой IDE
- `recipes/*.json` — карточки типов + `_schema.md`
- `templates/*` — код-шаблоны
- `conventions/*` — что класть в созданный проект
