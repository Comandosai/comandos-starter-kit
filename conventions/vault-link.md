# Привязка проекта к базе знаний (для агента)

Если `config.knowledge_base` НЕ null — заведи в базе знаний запись о проекте,
чтобы знания компаундились (по методологии COMANDOS / Karpathy LLM Wiki).

## Если база знаний — это COMANDOS-vault (есть папка `01-Projects/`)
1. Создай папку `01-Projects/NN-<имя-проекта>/` (NN — следующий номер).
2. Положи туда `README.md`: что за проект, стек, ссылка на код-репо, статус.
3. Добавь строку в корневой `log.md`:
   `create | <тип> «<имя>» → [[01-Projects/NN-<имя>/README]] #starter-kit`

## Если база знаний — произвольная папка
1. Создай `<knowledge_base>/projects/<имя-проекта>.md`.
2. Опиши: что, зачем, где код, статус, дата.

## Ветка A — разовый артефакт (презентация/видео/картинка)
Не заводи папку проекта `01-Projects/NN-...`. Достаточно лёгкой заметки:
- COMANDOS-vault → строка в `log.md`:
  `create | артефакт «<имя>» (<тип>) → <путь к файлу> #starter-kit #артефакт`
  (при желании короткая заметка в `00-Inbox/`/подходящей папке).
- произвольная база → файл `<knowledge_base>/artifacts/<имя>.md` со ссылкой на результат.

## Если базы знаний нет (null)
Ничего не делаем — проект standalone. При желании предложи резиденту
настроить «Второй мозг» из курса COMANDOS позже.

## Принцип
База знаний хранит ЗНАНИЕ о проекте (что это, решения, статус), а не код.
Код живёт в своём репо. Тяжёлые медиа — отдельно (§17). Ссылки связывают всё.
