# COMANDOS Starter — оркестратор

Главный сценарий. Подробный пошаговый рулбук — в `bootstrap.md`. Здесь — карта.

## Поток (5 фаз)

```
DOCTOR → CONFIG+VAULT → ИНТЕРВЬЮ → РЕЦЕПТ → ФИНАЛИЗАЦИЯ
```

1. **DOCTOR** (`doctor/doctor.sh`) — окружение, IDE, кандидаты базы знаний.
2. **CONFIG+VAULT** — `~/.comandos/config.json` (создать если нет, 3 вопроса один раз);
   найти/привязать базу знаний или предложить «Второй мозг» / папку.
3. **ИНТЕРВЬЮ** — новый/существующий проект + что делаем → определить ветку (A/B/C)
   и подобрать рецепт (`recipes/`). Точного нет → «Открытый путь» (`recipes/_schema.md`).
4. **РЕЦЕПТ** — развернуть: `template` (копия папки) / `repo` (degit) /
   `action:skill|mcp` (запуск навыка/MCP). Перед этим doctor доставит `needs`
   (CLI/MCP/env — см. `lib/mcp-install.md`).
5. **ФИНАЛИЗАЦИЯ** — положить `AGENTS.md` + `PROJECT-BRIEF.md` (`conventions/`),
   привязать к базе знаний (`conventions/vault-link.md`), показать как запустить.

## Правила
- Резидент — НЕ программист. Никаких стектрейсов. Всё делать за него, объяснять просто.
- TypeScript по умолчанию для продуктов. Не ломать существующую структуру.
- Convention, not catalog: рецепты — дефолты, а не ограничение. Хочет иное — собрать по правилам.

## Файлы kit
- `bootstrap.md` — полный пошаговый сценарий (читать первым)
- `doctor/doctor.sh` — проверка окружения
- `lib/mcp-install.md` — установка MCP/CLI по каждой IDE
- `recipes/*.json` — карточки типов + `_schema.md`
- `templates/*` — код-шаблоны
- `conventions/*` — что класть в созданный проект
