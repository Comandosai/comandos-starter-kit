# COMANDOS Starter — оркестратор

Главный сценарий. Подробный пошаговый рулбук — в `bootstrap.md`. Здесь — карта.

## Поток (фазы)

```
DOCTOR → CONFIG+VAULT → ИНТЕРВЬЮ → РЕЦЕПТ → СБОРКА → ПРОВЕРКА → ПРИВЯЗКА → ИТОГ
```

1. **DOCTOR** (`doctor/doctor.sh` / `.ps1`) — окружение, IDE, версия Node, кандидаты базы знаний.
2. **CONFIG+VAULT** — `~/.comandos/config.json` (создать если нет); найти/привязать базу
   знаний; предупредить если `projects_dir` в облаке (iCloud/Dropbox — git/npm не работают).
3. **ИНТЕРВЬЮ** — новый/существующий проект + что делаем → ветка (A/B/C) + рецепт.
4. **РЕЦЕПТ** — развернуть: `template`/`repo`/`scaffold_cmd`/`action:skill|mcp`; доустановить `needs`.
5. **СБОРКА** — полный набор файлов в проект (`conventions/`): `AGENTS.md`+`CLAUDE.md`+
   `.agent/rules`+`PROJECT-BRIEF.md`+`.gitignore`+**рабочий `.env`**; подставить все `{{...}}`.
6. **ПРОВЕРКА** (gate) — `git init`, `npm install`, проверка типов, smoke-запуск. Не говорить
   «готово», пока не зелёное.
7. **ПРИВЯЗКА** — обязательно создать заметку о проекте в базе знаний (`conventions/vault-link.md`).
8. **ИТОГ** — что создано, результат проверки, как запустить, что дальше.

Полная самопроверка — блок «Definition of Done» в конце `bootstrap.md`.

## Правила
- Резидент — НЕ программист. Никаких стектрейсов. Всё делать за него, объяснять просто.
- TypeScript по умолчанию для продуктов. Не ломать существующую структуру.
- Convention, not catalog: рецепты — дефолты, а не ограничение. Хочет иное — собрать по правилам.

## Файлы kit
- `bootstrap.md` — полный пошаговый сценарий (читать первым)
- `doctor/doctor.sh` — проверка окружения
- `lib/mcp-install.md` — установка MCP/CLI по каждой IDE
- `recipes/*.json` — карточки типов + `_schema.md`
- `templates/*` — код-шаблоны
- `conventions/*` — что класть в созданный проект
